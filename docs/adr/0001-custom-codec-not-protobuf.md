# ADR 0001 — Custom AST-generated codec, not Protobuf / Avro / msgp

- Status: Accepted
- Date: 2026-07-08
- Scope: Gothic v3.0.0 WASM re-architecture, Part II
- Related: [0002](0002-schema-seam-generic-interpreter-deferred.md), [0003](0003-two-tier-protocol.md), [0004](0004-static-full-go-core.md)

## Context

Gothic Topics transport per-field reactive UI state between independent WASM
components through the JS event bus as little-endian binary frames. The wire
codec is generated by the CLI directly from the user's Go struct in
`src/topics/` (see `internal/build/wasm_codec.go`): for each field it emits an
`_encode_X` / `_decode_X` line pair, plus `_capture_X` for the manager's
per-field zero-copy path. The runtime and server sides of that codec live in
`wasm/wasm-runtime/runtime/codec.go`, its byte-identical host copy in
`wasm/internal/parity/codec_runtime.go`, and the user-facing stub in
`wasm/stubs.go`.

The recurring question is whether we should stop hand-rolling this and adopt an
established serialization framework — Protocol Buffers, Apache Avro, or a
MessagePack library such as `tinylib/msgp`. The appeal is "battle-tested":
someone else owns the wire format and its edge cases.

Constraints that make this decision non-obvious:

- **TinyGo toolchain.** Components compile with TinyGo to keep WASM binaries
  tiny. Protobuf's canonical Go runtime (`google.golang.org/protobuf`) leans on
  reflection and a large runtime that inflates binary size and does not sit
  comfortably on TinyGo. Reflection-heavy codecs are exactly what we are trying
  to avoid on the client.
- **Binary size is a product feature.** Every KB of WASM is download + parse +
  linear-memory cost. The generated codec is straight-line append/read code with
  zero reflection and no schema registry in the binary.
- **The struct is the source of truth (DX).** A Gothic user writes a plain Go
  struct and adds `gothic:` tags. Adopting `.proto` / `.avsc` would force a
  second schema artifact and a codegen step the user has to keep in sync — a
  regression in the "just write Go" developer experience.
- **Payloads are small UI structs.** Topic payloads are UI state — bytes to low
  kilobytes. Protobuf's tag-per-field varint framing is designed for
  forward/backward-compatible RPC messages and is *larger* than our tagless
  positional frame for these small structs, where both sides already agree on
  the field order via the shared generated schema.
- **The real risk is drift, not format.** The genuine failure mode of a custom
  codec is encoder ↔ decoder divergence (or client ↔ server divergence). That is
  a *testing* problem, and it is solvable without a third-party format.

## Decision

Keep the custom AST-generated, tagless, positional binary codec. Do **not**
adopt Protobuf, Avro, or msgp as the wire format.

Kill the drift risk with tests rather than a library. The suite adds, in
`wasm/wasm-runtime/runtime/`:

- `codec_roundtrip_test.go` — executable round-trip coverage of **every**
  supported wire type (primitives, type aliases, slices, nested slices, maps,
  nested maps, nil/non-nil pointers, nested structs, `time.Time`, and every
  `gothic:` tag including `skip`), asserting `reflect.DeepEqual` and the
  WireVersion header byte. The encode/decode closures are hand-matched to the
  codegen shapes, so they pin wire *semantics* while `TestCodecGolden` pins the
  generated *strings* and `codec_parity_test.go` pins stub↔runtime byte-parity.
- `codec_fuzz_test.go` — `FuzzCodecRoundTrip` (encode→decode identity) and
  `FuzzDecoderNeverPanics` (arbitrary/truncated/garbage/wrong-version input
  never panics and always sets `Decoder.Err`).

We adopt the good **ideas** from the surveyed formats without adopting their
formats:

- **Avro's "tagless binary + external schema".** Our frames are tagless; the
  "external schema" is the generated codec plus the schema descriptor
  seam (see ADR 0002). This is where the size win comes from.
- **Protobuf varint / zigzag.** Reserved as a *future* per-field optimization if
  profiling ever shows the fixed-width integers dominate a real payload. Not
  adopted pre-emptively (YAGNI); today fixed-width keeps encode/decode
  branch-free.
- **Schema fingerprinting for skew.** Avro's schema-hash idea is realized as the
  `schemaId` content hash, so version skew is *detectable* even though
  nothing interprets the schema yet.

## Consequences

- We own the wire format and its correctness. The round-trip + fuzz suite is now
  the contract; it must stay green and must grow a case whenever a new type kind
  becomes encodable.
- Any change to `codec.go` must be mirrored into `internal/parity/codec_runtime.go`
  **and** `stubs.go` — enforced by `TestFilesInSyncWarning` and the parity tests.
- Binaries stay small and reflection-free on TinyGo; no schema registry ships in
  the WASM.
- We forgo Protobuf/Avro's mature cross-language tooling. Acceptable: the wire is
  an internal Gothic transport between Gothic-generated peers, not a public
  contract.
- If a future requirement demands cross-language consumers or on-wire schema
  evolution, revisit — the WireVersion byte already reserves the
  break mechanism, and the schema seam (ADR 0002) reserves the descriptor.
